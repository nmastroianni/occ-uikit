Button

Create stout buttons with just one class.

Usage

<button class="uk-button uk-button-default">Button</button>

Example:

Component Options

Class Description
.uk-button-default Creates a default, themeable button style.
.uk-button-primary Indicates the primary action, giving it a more prominent look.
.uk-button-secondary Indicates a secondary action, giving it a less prominent look.
.uk-button-danger Indicates a destructive or dangerous action, typically styled in red.
.uk-button-text Removes button styling, making it look like a plain text link.
.uk-button-link Mimics the appearance of a hyperlink, often used for less critical actions.
.uk-button-small Reduces the button size for a more compact appearance.
.uk-button-large Increases the button size for greater prominence.
.uk-width-1-1 Makes the button span the full width of its container.
disabled Disables the button, making it unclickable and visually muted.

Accessibility

The Button component is designed with accessibility in mind, ensuring that all users can interact with it effectively. Here are some key considerations:

  • Keyboard Navigation: All buttons are focusable and can be activated using the Enter or Space keys, ensuring full operability for keyboard-only users.
  • Screen Reader Support: Buttons are properly announced by screen readers, and the disabled attribute is correctly interpreted, preventing interaction with inactive buttons.
  • Clear Labeling: It is crucial to provide clear and descriptive text within the button to inform users of its purpose. Avoid generic labels like "Click Here."
