Accessibility

The Card component is designed with accessibility in mind, ensuring that all users can interact with it effectively. Here are some key considerations:

  • Semantic Structure: Use proper semantic HTML for the content within the card. For example, use headings <h1>-<h6> for titles and <p> for text.
  • Heading Hierarchy: Ensure that the heading hierarchy is logical and consistent. The .uk-card-title should be an appropriate heading level for the context in which the card is used.
  • Image Alt Text: If the card contains an image, make sure to provide a descriptive alt attribute for the image to describe its content to screen reader users.
