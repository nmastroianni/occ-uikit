Accessibility

The Comment component is designed with accessibility in mind. Here are some key considerations:

  • Semantic Structure: Use proper semantic HTML for the content within the comment. For example, use headings <h1>-<h6> for titles and <p> for text.
  • Heading Hierarchy: Ensure that the heading hierarchy is logical and consistent. The comment title should be an appropriate heading level for the context in which the comment is used.
  • Image Alt Text: If the comment contains an image, make sure to provide a descriptive alt attribute for the image to describe its content to screen reader users.
